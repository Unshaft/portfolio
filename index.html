Projet 1: Tableau de bord R Shiny

Contexte du projet :

L'objectif de ce projet était de créer une application web interactive pour suivre et analyser les données géographiques des infrastructures sportives en France. Les données proviennent du site officiel de l'État et comprennent des informations détaillées sur chaque infrastructure, telles que les coordonnées géographiques, l'année de construction, le public cible, la disponibilité d'un site internet et l'accessibilité pour les personnes à mobilité réduite.

Objectif principal :

Faciliter l'accès et la visualisation des données des infrastructures sportives pour les utilisateurs, permettant des analyses spécifiques et une meilleure compréhension de la répartition et des caractéristiques des infrastructures sportives en France.

Fonctionnalités principales :

  • Sélection de la ville : Les utilisateurs peuvent choisir une ville en France pour afficher les infrastructures sportives disponibles. Les villes sont sélectionnées dynamiquement en fonction de la région et du département choisis.
  • Filtres avancés :
    • Année de construction : Permet de filtrer les infrastructures en fonction de leur année de construction.
    • Public cible : Filtre les infrastructures par public cible (jeunes, adultes, seniors).
    • Accessibilité : Filtre les infrastructures accessibles aux personnes à mobilité réduite.
    • Type de propriétaire : Filtre par type de propriétaire (commune, département, région, etc.).
    • Type de sol : Filtre par type de surface de sol (bitume, gazon naturel, synthétique, etc.).
    • Nature de l'équipement : Filtre par nature de l'équipement (intérieur, extérieur, site naturel, etc.).
  • Visualisation des données :
    • Carte interactive Leaflet : Affiche les infrastructures sportives sous forme de marqueurs géolocalisés. Les utilisateurs peuvent cliquer sur les marqueurs pour obtenir des informations détaillées.
    • Carte choroplèthe : Affiche une carte colorée représentant la densité des infrastructures par région, département ou commune.
    • Tableau des données : Affiche les infrastructures dans un tableau interactif avec des options de recherche et de filtrage.
    • Graphiques dynamiques avec Plotly :
      • Histogramme des types d'équipements.
      • Répartition des équipements par type de propriétaire.
      • Accessibilité des infrastructures.
      • Répartition des équipements par type de sol.
      • Nombre d'équipements par nature de site.
      • Nombre d'équipements par type d'utilisateur.
      • Évolution du nombre d'installations au fil des années.

Déroulement du projet :

Collecte des données : Importation des données des infrastructures sportives à partir du site officiel de l'État via une API REST. Utilisation de httr et jsonlite pour gérer les requêtes API et le traitement des réponses JSON.

Prétraitement des données : Nettoyage et formatage des données pour garantir leur qualité et leur compatibilité avec l'application Shiny. Sélection des colonnes pertinentes et gestion des valeurs manquantes.

Développement de l'application : Utilisation de R Shiny pour le développement de l'application web. Intégration de la bibliothèque Leaflet pour la visualisation des cartes. Utilisation de Plotly pour la création de graphiques interactifs. Utilisation de DT pour les tableaux interactifs. Application de styles personnalisés pour améliorer l'esthétique de l'application.

Fonctionnalités techniques détaillées :

Interface utilisateur (UI) : Utilisation de shinydashboard pour structurer l'interface utilisateur avec des onglets pour l'introduction, le tableau, les graphiques et la carte. Inclusion de filtres interactifs pour affiner la recherche des infrastructures.

Serveur (Server) : Gestion des requêtes API pour récupérer les données des infrastructures sportives. Traitement des données récupérées pour les afficher dans les différents composants de l'interface utilisateur. Application des filtres sélectionnés par l'utilisateur pour mettre à jour dynamiquement les visualisations et les tableaux. Génération de rapports téléchargeables au format PDF avec rmarkdown.

Visualisation des données :

  • Carte interactive Leaflet : Affichage des infrastructures sportives sous forme de marqueurs géolocalisés. Utilisation de clusters de marqueurs pour une meilleure visualisation des points de données. Ajout de couches de carte différentes (Default, Light, Satellite).
  • Carte choroplèthe : Génération de cartes choroplèthes pour représenter la densité des infrastructures par région, département ou commune. Utilisation de palettes de couleurs pour indiquer la densité des infrastructures.
  • Graphiques Plotly : Création de graphiques interactifs pour afficher les données agrégées par type d'équipement, type de propriétaire, accessibilité, type de sol, nature de l'équipement, type d'utilisateur et année de création.

Téléchargement des données : Option de téléchargement des données filtrées sous forme de fichier JSON. Génération de rapports PDF téléchargeables avec des graphiques et des statistiques résumant les données sélectionnées.

Projet 2: Optimisation de la Maintenance des Véhicules de Transport Collectif

Contexte et Objectifs :

Contexte :

Dans le cadre de l'amélioration des opérations de maintenance de ses véhicules, une entreprise de transport collectif souhaite développer des recommandations assistées pour ses techniciens de maintenance. L'objectif est de mutualiser l'expertise des techniciens répartis sur différents sites géographiques et intervenant sur des équipements similaires.

Objectifs :

Le projet vise à créer des outils d'aide à la décision pour piloter la maintenance et réaliser les actions de réparation en utilisant des données issues de la GMAO (Gestion de Maintenance Assistée par Ordinateur). Les données sont anonymisées et confidentielles, et leur partage est strictement limité au cadre du projet.

Contributions Techniques :

Prétraitement des données :

  • Chargement et fusion de différentes sources de données.
  • Transformation des colonnes catégorielles et imputation des valeurs manquantes.

Modélisation :

  • Utilisation de DecisionTreeClassifier pour prédire les types de systèmes et les ordres de travail.
  • Évaluation des modèles de prédiction avec des métriques de précision.

Interface Utilisateur avec Dash :

  • Développement d'une interface interactive permettant de sélectionner les modalités et de visualiser les prédictions.
  • Intégration de graphiques interactifs pour afficher les résultats des modèles de prédiction.

Technologies Utilisées :

  • Python : Pandas, Scikit-learn pour le traitement des données et la modélisation.
  • Dash : Développement de l'interface utilisateur.
  • Plotly : Visualisation des données.
  • CSS : Personnalisation de l'apparence de l'application.

Résultats :

Le projet a permis de :

  • Centraliser et exploiter les données de maintenance.
  • Améliorer les recommandations pour les techniciens de maintenance.
  • Développer une interface utilisateur intuitive pour faciliter l'accès aux prédictions des modèles.

Conclusion :

Ce projet illustre l'application de techniques de data science et de développement d'interfaces utilisateur pour optimiser les opérations de maintenance dans le secteur des transports collectifs. Il démontre la capacité à transformer des données complexes en outils pratiques et accessibles pour les techniciens sur le terrain.
